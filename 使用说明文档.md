# 抽卡概率工具平台 - 使用说明

## 一、概述

本平台提供两种工作模式，用于测试和验证抽卡概率：

| 模式 | 说明 | 数据来源 |
|------|------|----------|
| **本地** | 纯浏览器端运算，使用 `data/cards.json` 定义的卡池 | 本地JSON文件 |
| **服务器** | 通过TCP RPC连接C#游戏服务器，调用真实抽卡接口 | 游戏服务器 |

> **注意：** 服务器模式的连接地址、账号密码等信息均在后台 `config.py` 中配置，  
> 前端不暴露任何连接参数，切换到"服务器"模式时自动连接并登录。

## 二、快速启动

### 1. 环境要求

- **Python 3.8+**
- **Flask** (`pip install flask`)
- 无其他第三方依赖

### 2. 启动命令

```bash
cd card_platform
python app.py
```

默认启动在 `http://localhost:5007`，浏览器自动打开。

### 3. 目录结构

```
card_platform/
├── app.py                    # Flask应用入口
├── config.py                 # 配置文件（端口、限制等）
├── data/
│   └── cards.json            # 本地卡池数据定义
├── routes/
│   ├── __init__.py
│   ├── gacha_routes.py       # 基础路由（主页、cards.json）
│   └── game_server_routes.py # 游戏服务器桥接路由
├── services/
│   ├── gacha.py              # 本地抽卡核心算法
│   └── game_rpc_client.py    # TCP RPC客户端
├── templates/
│   └── index.html            # 主页面模板
└── static/
    ├── css/style.css
    └── js/modules/           # 前端JS模块
```

## 三、本地模式使用

### 适用场景

- 不需要启动游戏服务器
- 快速验证卡池概率配置
- 大量抽卡压测（支持最多10万次）

### 使用步骤

1. 启动平台后，默认进入**本地模式**
2. 修改 `data/cards.json` 配置卡池（卡牌、概率、保底等）
3. 点击 `🔄 刷新数据` 重新加载配置
4. 选择卡池，点击抽卡按钮进行测试

### cards.json 配置说明

```json
{
  "pools": [
    {
      "pool_id": "standard",
      "name": "标准池",
      "description": "标准卡池",
      "cards": [
        {
          "card_id": "SSR_001",
          "name": "角色名",
          "rarity": "SSR",
          "weight": 6,
          "is_featured": true
        }
      ],
      "pity": {
        "soft_pity_start": 74,
        "hard_pity": 90,
        "soft_pity_increment": 6.0
      }
    }
  ]
}
```

**关键字段：**

| 字段 | 说明 |
|------|------|
| `weight` | 抽取权重，同稀有度内的相对概率 |
| `is_featured` | 是否为UP角色 |
| `soft_pity_start` | 软保底触发起始抽数 |
| `hard_pity` | 硬保底（必出）抽数 |
| `soft_pity_increment` | 每超过软保底1抽增加的概率(%) |

## 四、游戏服务器模式使用

### 适用场景

- 测试C#游戏服务器的真实抽卡接口
- 验证服务端保底逻辑和概率
- 对接 XActor RPC 框架

### 前提条件

1. **C#游戏服务器已启动并监听TCP端口**
2. 确认服务器实现了以下RPC接口：
   - `ILoginServer.Login(name, password)` — 登录
   - `IPlayerGachaServer.GetCardPoolList()` — 获取卡池列表
   - `IPlayerGachaServer.GachaCardPool(poolId, times)` — 抽卡
   - `IPlayerGachaServer.ReadCardPoolNew(poolId)` — 标记新卡已读

### 使用步骤

1. 在 `config.py` 中配置游戏服务器信息（见下方"后台配置"）
2. 启动平台后，点击右上角 **服务器** 按钮切换模式
3. 平台自动连接游戏服务器并使用配置的账号登录
4. 连接成功后自动加载卡池列表
5. 选择卡池，进行抽卡操作

### 后台配置

在 `config.py` 末尾的 `GAME_SERVER_CONFIG` 中调整（也可通过环境变量覆盖）：

```python
GAME_SERVER_CONFIG = {
    'host': '127.0.0.1',      # 游戏服务器IP    (环境变量 GAME_HOST)
    'port': 8001,              # 游戏服务器TCP端口 (环境变量 GAME_PORT)
    'timeout': 10,             # 连接超时(秒)     (环境变量 GAME_TIMEOUT)
    'username': 'test',        # 登录账号         (环境变量 GAME_USER)
    'password': '',            # 登录密码         (环境变量 GAME_PASS)
}
```

也可以通过环境变量启动：

```bash
GAME_HOST=192.168.1.100 GAME_PORT=9001 GAME_USER=admin GAME_PASS=123 python app.py
```

> **安全提示：** 生产环境建议通过环境变量传入账号密码，避免明文写在配置文件中。

## 五、注意事项

### 通用注意事项

1. **模式切换会重置当前抽卡数据**，切换前请先导出需要的数据
2. **导出数据**功能仅在本地模式可用，游戏服务器模式暂不支持
3. 大量抽卡（>500次）时会显示进度条，请耐心等待

### 本地模式注意事项

1. 修改 `cards.json` 后**必须点击刷新数据**才会生效
2. 抽卡数据存储在浏览器 Session 中，关闭页面后丢失
3. 最大单次抽卡限制为 **100,000 次**
4. 卡池概率权重是**相对值**，同稀有度内的卡牌按权重分配

### 游戏服务器模式注意事项

1. 切换到服务器模式时**自动使用后台配置的账号连接**，无需手动输入
2. 连接超时默认 10 秒，可在 `config.py` 的 `GAME_SERVER_CONFIG` 中调整
3. 抽卡调用的是真实服务端接口，**会消耗服务端资源**
4. 十连抽对应 `EnumGachaTimes.Ten`，单抽对应 `EnumGachaTimes.One`
5. 如果服务器断开连接，**重新切换到服务器模式**会自动重连
6. 卡池列表由服务器返回，不支持本地自定义
7. 前端**不暴露服务器地址和账号信息**，全部在后台 `config.py` 管理

### 通信协议说明

游戏服务器通信使用自定义 TCP RPC 协议：

```
帧格式：[4字节小端int32长度] + [UTF-8 JSON数据]

消息信封：BasicMsg {
    rpcId: string,       // RPC接口标识（如 "ILoginServer.Login"）
    returnId: int,       // 回调ID，用于匹配请求和响应
    msg: byte[]          // 参数/返回值，JSON序列化后的字节数组
}
```

> **重要**：`msg` 字段的序列化格式取决于 C# ProtocolGenerator 生成的代码。
> 当前 Python 客户端假设参数以 JSON 数组形式编码。
> 如果实际服务器使用不同编码，需要修改 `services/game_rpc_client.py` 中 `_send_rpc` 方法的序列化逻辑。

## 六、开发与扩展

### 添加新卡池（本地模式）

编辑 `data/cards.json`，在 `pools` 数组中添加新的卡池对象。

### 修改服务器端口

编辑 `config.py`：

```python
SERVER_CONFIG = {
    'host': '0.0.0.0',
    'port': 5007,        # Web平台端口
    'debug': True
}
```

### 自定义RPC接口

如需对接新的服务器接口，修改以下文件：

1. `services/game_rpc_client.py` — 添加新的RPC调用方法
2. `routes/game_server_routes.py` — 添加新的Flask桥接路由
3. `static/js/modules/api.js` — 添加前端API调用
4. `static/js/modules/gacha-engine.js` — 添加业务逻辑

### 文件修改影响范围

| 修改文件 | 影响 |
|----------|------|
| `config.py` | 游戏服务器地址/账号/端口配置 |
| `data/cards.json` | 本地模式卡池配置 |
| `services/game_rpc_client.py` | 游戏服务器通信 |
| `routes/game_server_routes.py` | HTTP→TCP桥接接口 |
| `static/js/modules/gacha-engine.js` | 前端抽卡逻辑核心 |
| `static/js/modules/ui.js` | 界面显示更新 |

## 七、故障排查

| 问题 | 解决方案 |
|------|----------|
| 启动报 `ModuleNotFoundError: flask` | 执行 `pip install flask` |
| 本地模式无卡池 | 检查 `data/cards.json` 是否存在且格式正确 |
| 服务器模式连接失败 | 检查 `config.py` 中 `GAME_SERVER_CONFIG` 的 host/port 是否正确，服务器是否已启动 |
| 登录后无卡池列表 | 确认服务器实现了 `GetCardPoolList` 接口 |
| 抽卡返回错误码 | 检查服务器日志，可能是协议序列化格式不匹配 |
| 服务器模式显示"未连接" | 刷新页面重新自动连接，或检查后台配置 |
| `msg` 字段解析失败 | 检查 `game_rpc_client.py` 中的序列化方式是否与C#端一致 |
